# 🈺 rfq-to-quotation（译盘）— 询盘转中文报价表

一个 Claude Code skill：把客户发来的英文/俄文询盘 Excel，转换成供内部报价员填写单价的**中文报价表**。

## 为什么做这个

外贸业务的日常痛点：客户发来一份英文或俄文询盘 Excel，要转给内部报价员填单价——但报价员看不懂外语列头和产品名，业务员得先人肉翻译列头、翻译产品名称，遇到只写规格号（如 `M8` / `D8`）没写具体尺寸的行，还要一条条去查标准尺寸表补全。一单询盘几十行，翻译 + 查表下来常常要一小时，还容易翻译不一致或漏查。

## 核心亮点

**动态列识别，信息零丢失**：很多客户的 Excel 里会有"空列头但有数据"的列——比如某一列没写列头，但整列都是 `C75S`、`65Mn`、`SS304` 这类值。人工处理时这类列经常被直接忽略或误删。这个 skill 会扫描全部列（不管列头是否存在），对无列头但有数据的列按内容特征推断含义（含 `C75S`/`A2`/`65Mn`/`SS304` → 钢材牌号；含 `ZP`/`HDG`/`BZP` → 表面处理），推断不出来也会保留数据并标注"未命名列"——**原则是宁可留给人工确认，也绝不丢数据**。

其他能力：
- 列头 + 产品名称中英/俄对照翻译
- 只有规格号的行自动查 DIN 标准尺寸表补全（内径×外径×厚度 / 轴径×外径×厚度等）
- 生成的报价表按 DIN 标准分组、报价列自动标黄，报价员打开就能填

## 输出

```
询盘 [编号] — [产品类型] 报价表（[标准列表]）
─────────────────────────────────────────
产品描述 | 尺寸规格(mm) | ...询盘原有列（中文列头）... | 报价(CNY/KG) ★黄色
```

## 先跑个 demo

```bash
python3 examples/generate_demo.py
```

生成 `examples/demo_inquiry.xlsx`，9 行虚构询盘数据，覆盖 DIN 125/7980/471 三个标准，并故意留了一列无列头的钢材牌号数据用于验证动态识别。详见 [`examples/README.md`](./examples/README.md)。

## 安装

把 `rfq-to-quotation/` 目录放进 Claude Code skills 目录（如 `~/.claude/skills/rfq-to-quotation/`），或对 Claude 说"帮我安装这个 skill：<仓库URL>"。

依赖：`python3` + `xlrd`（读 .xls）+ `openpyxl`（读写 .xlsx）。

## 适用范围

本 skill 输出**中文报价表（内部使用）**，用于报价员填写单价。生成发给客户的英文报价单不在本 skill 范围内。
