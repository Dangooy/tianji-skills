# 评分规则与邮箱分级

S4 评分阶段用。核心原则：**确定性打分，可复现**——同样的输入必得同样的分数和分级。这替代"LLM 主观一句话 A/B/C"，让线索质量可解释、可审计。

---

## 邮箱有效性分级（E1–E4）

无付费邮箱验证器的前提下，按**来源可信度**分级，直接决定"能不能发开发信"：

| 级别 | 判据 | 处置 | 打分权重值 |
|------|------|------|-----------|
| **E1 高** | 官网明示的**具名**或**部门**邮箱（如 `j.smith@co.com` 或 `purchasing@co.com`），域名 = 官网域名 | 可直接发开发信 | 1.0 |
| **E2 中** | 官网 **generic** 邮箱（`info@`/`sales@`/`office@`），域名匹配 | 可发，用通用触达模板 | 0.7 |
| **E3 推断** | 按域名 + 常见模式推断（`firstname@domain`），**未在任何页面出现** | 标注"推断·未验证"，仅备选 | 0.3 |
| **E4 无** | 只有联系表单/电话/社媒，无可用邮箱 | 走非邮件渠道，不算可触达 | 0.0 |

**可选免费增强**：`dig MX <domain> +short` 检查域名是否有 MX 记录（能收邮件）。无 MX → E3 直接降 E4（域名都不收信，推断邮箱无意义）。这只证明域名收信，不证明具体邮箱存在。

## 确定性加权打分

> 以下权重是**通用默认值**，请按你的行业和触达策略调整（例如重展会跟进的行业可提高决策人分层权重）。调整后保持"确定性、可复现"即可。

```
score = 0.40 · 邮箱可触达分     (E1=1.0 / E2=0.7 / E3=0.3 / E4=0.0，取该公司最佳邮箱)
      + 0.25 · 交叉验证分       (min(cross_source_count, 3) / 3，即 ≥3 独立源=1.0)
      + 0.20 · 决策人分层分     (具名决策人=1.0 / 部门角色=0.6 / 仅generic=0.3 / 无=0.0)
      + 0.15 · 规模匹配分       (规模信号强度 × 产品线重合度，见下)
      − 0.20 · 疑点惩罚         (每个 red_flag 扣 0.1，最多扣 0.2)

最终 score 截断到 [0, 1]。
```

**规模匹配分**（0–1）：粗粒度即可
- 规模信号（员工数/经营年限/分支/社媒活跃）齐全且是分销商/批发商 → 0.8–1.0
- 部分信号 → 0.4–0.7
- 几乎无信号（只有个官网） → 0.1–0.3
- 乘以产品线重合度（你的产品在其采购/经销范围内的匹配程度，0–1，产品线见 `config/company-profile.md`）

**交叉验证源桶**（每个独立桶算 1 个 source）：
① 公司官网（team/about/contact 页）② 行业目录/黄页（Europages/Kompass/本地）③ 母语媒体/招标/新闻 ④ 社媒公开页（LinkedIn公司页/VK，只读）⑤ 已知同业旁证（同城同品类）。**单源 = 天然 C 级**。

---

## 分级（tier）

```
🟢 A 「可直接发开发信」   score ≥ 0.75  且  最佳邮箱 ∈ {E1, E2}  且  cross_source_count ≥ 2
🟡 B 「触达前需补一步验证」 0.5 ≤ score < 0.75  或  最佳邮箱 = E3
🔴 C 「仅供参考·待核实」   score < 0.5  或  单源  或  最佳邮箱 = E4
```

## 诚实原则（硬约束，写进报告）

- 系统**永不承诺**海关采购记录、决策人私人手机/私邮——这些需付费专业库，本流程拿不到。
- **C 级线索必须带 `next_verification_step`**（如"需人工核实采购负责人姓名"/"需电话确认邮箱有效"），不伪装成可触达。
- 报告元数据 Sheet 必须含免责声明："本报告基于公开网页信息，不含海关采购记录，联系方式可信度见各行 email_level 分级。"
